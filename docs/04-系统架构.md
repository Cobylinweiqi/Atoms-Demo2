## 4.3 关键架构决策

### ① 虚拟文件系统（VFS）

- 项目文件存储在 PostgreSQL 中，非真实磁盘
- 每个文件有版本历史（类似 Git）
- Sandbox 服务从 VFS 读取文件 → 挂载到容器 → 运行 Vite Dev Server
- 好处：云端持久化、多端同步、无需管理服务器磁盘

### ② AI 编排器独立服务

- 专注于 LLM 调用、流式输出、工具执行
- 可独立扩缩（AI 是最重负载的服务）
- 对外暴露 SSE 端点 + 内部 gRPC

### ③ 事件驱动架构

- 所有状态变更发布到 Redis Pub/Sub
- Realtime Gateway 订阅事件 → 推送 WebSocket
- 允许松耦合扩展（未来加分析、通知等服务）

### ④ 多租户隔离

- 组织（Organization）为租户边界
- API 层强制 org_id 过滤
- 数据库层可选 Row-Level Security
- Sandbox 容器按项目隔离

### ⑤ 安全模型

- Row-Level Security（PostgreSQL RLS）实现多租户隔离
- API Key 使用 AES-256-GCM 加密存储
- 聊天端点速率限制（防滥用）
- 内容审核（敏感词过滤）
- 沙箱代码执行 Docker 隔离
- CSP / CORS 安全头

### ⑥ 协作模型

- Yjs（CRDT）实现冲突无关协同编辑
- 在线状态（Presence）通过 WebSocket
- 光标共享（Cursor Sharing）
- 评论/标注系统（未来）

## 4.4 数据流详解

### AI 对话数据流

```
用户输入消息
    │
    ▼
[Chat UI] ──HTTP POST──▶ [BFF Route Handler]
                                │
                                ▼
                    [AI Orchestrator Service]
                                │
                    ┌───────────┼───────────┐
                    ▼           ▼           ▼
              [Context     [Prompt      [Streaming
               Builder]    Templates]   Engine]
                    │
                    ▼
              [LLM Provider] (OpenAI/Anthropic/...)
                    │
                    ▼
              [Tool Executor] ──▶ [VFS Service] ──▶ PostgreSQL
                    │                    │
                    │                    ▼
                    │              [Event Bus (Redis Pub/Sub)]
                    │                    │
                    │                    ▼
                    │              [Realtime Gateway] ──▶ [WebSocket]
                    │                                          │
                    ▼                                          ▼
              [SSE Stream] ◀────────────────────────── [Chat UI]
```

### 实时预览数据流

```
文件变更 (AI 工具 / 用户编辑)
    │
    ▼
[VFS Service] ──写入──▶ [PostgreSQL]
    │
    ├──发布事件──▶ [Redis Pub/Sub] ──▶ [Realtime Gateway] ──▶ [WebSocket]
    │                                                         │
    └──通知──▶ [Sandbox Service]                              ▼
                    │                                   [Preview UI]
                    ▼                                   (iframe 刷新)
              [Docker Container]
              (Vite Dev Server + HMR)
                    │
                    ▼
              [iframe 自动更新]
```

### 部署数据流

```
用户点击部署
    │
    ▼
[Deploy UI] ──HTTP POST──▶ [Deploy API]
                                │
                                ▼
                        [Sandbox Service]
                        (npm install + build)
                                │
                                ▼
                        [Upload Artifacts] ──▶ [S3/MinIO]
                                │
                                ▼
                        [Deploy Provider]
                        (Vercel/Netlify/Docker)
                                │
                    ┌───────────┼───────────┐
                    ▼           ▼           ▼
              [DNS/SSL]   [CDN Config]  [Deploy Record]
                    │                        │
                    └──────────┬─────────────┘
                               │
                    [Redis Pub/Sub]
                               │
                               ▼
                    [Realtime Gateway] ──▶ [WebSocket] ──▶ [Deploy UI]
                    (deploy:progress / deploy:complete)
```

## 4.5 服务间通信

| 通信方式 | 场景 | 技术 |
|---|---|---|
| HTTP REST | 客户端 ↔ BFF ↔ Core API | Axios / fetch |
| SSE (Server-Sent Events) | AI 流式响应 | EventSource |
| WebSocket | 实时协作、推送 | Socket.io |
| Redis Pub/Sub | 服务间异步事件 | Node Redis |
| gRPC (未来) | 服务间高性能调用 | @grpc/grpc-js |
