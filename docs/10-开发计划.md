## 10.2 Phase 1: 基础设施搭建（Week 1-4）

| 周 | 任务 | 交付物 |
|---|---|---|
| W1 | Monorepo 初始化 | Turborepo + pnpm workspace、ESLint/Prettier/Husky 配置、CI 基础 |
| W1 | Docker 开发环境 | `docker-compose.yml`（PostgreSQL + Redis + MinIO）、热重载 |
| W1 | Prisma schema | 完整数据库 schema + 初始迁移 + 种子数据 |
| W2 | 设计系统 | Tailwind v4 配置、shadcn/ui 组件库（30+ 基础组件）、图标库 |
| W2 | 认证系统 | JWT + Refresh Token、GitHub/Google OAuth、注册/登录页面 |
| W3 | 组织/项目模块 | 组织 CRUD、成员管理、项目 CRUD、项目模板系统 |
| W3 | 文件模块 | 虚拟文件系统、文件树、版本历史 |
| W4 | 工作区框架 | 工作区 Shell、标签页系统、侧边栏、底部面板、命令面板（Cmd+K） |
| W4 | 控制台首页 | 仪表盘、项目列表（卡片/列表视图）、新建项目向导 |

## 10.12 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| AI 模型 API 变更/限流 | 核心功能不可用 | 多模型备选、本地模型兜底、缓存策略 |
| 沙箱容器安全风险 | 代码执行逃逸 | Docker 隔离、资源限制、网络隔离、定期安全审计 |
| Token 成本超预期 | 财务压力 | 配额管理、智能路由（简单任务用便宜模型）、上下文压缩 |
| Code↔Visual 双向同步技术难度高 | 编辑体验差 | 分阶段实现（先 Code→Visual，再 Visual→Code）、AST 精确修改 |
| 实时协作冲突 | 数据丢失 | Yjs CRDT、操作转换、版本回退 |
| 大项目文件数过多 | 性能下降 | 虚拟滚动、懒加载、文件分页、WebWorker 处理 AST |
