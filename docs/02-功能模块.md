## 2.1 AI Chat（AI 对话）

### 架构

```
┌──────────────────────────────────────────────────────────────────────┐
│                        AI Chat Module                                │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐ │
│  │  Chat UI         │    │  AI Orchestrator │    │  Tool System    │ │
│  │                  │───▶│                  │───▶│                  │ │
│  │  - 流式渲染      │    │  - 多模型路由    │    │  - 文件操作      │ │
│  │  - Markdown      │    │  - 上下文构建    │    │  - 组件创建      │ │
│  │  - 代码高亮      │    │  - Prompt 工程   │    │  - 主题应用      │ │
│  │  - 消息历史      │    │  - 流式响应      │    │  - 依赖安装      │ │
│  │  - 模型切换      │    │  - Token 统计    │    │  - 预览刷新      │ │
│  │  - 附件上传      │    │                  │    │  - 路由创建      │ │
│  │  - 工具调用展示  │    │                  │    │  - API 创建      │ │
│  │  - 停止/重新生成 │    │                  │    │  - 错误修复      │ │
│  └─────────────────┘    └──────────────────┘    └─────────────────┘ │
└──────────────────────────────────────────────────────────────────────┘
```

### 上下文构建策略

```
1. System Prompt (角色 + 项目类型 + 技术栈规范)
2. Conversation History (最近 N 轮对话)
3. Project Context (文件树结构 + 关键文件内容)
4. Active Component (当前编辑的组件)
5. Active Theme (当前主题配置)
6. User Message
```

### AI 工具定义（Tool Definitions）

| 工具名 | 描述 | 输入 | 输出 |
|---|---|---|---|
| `create_file` | 创建新文件 | `{path, content}` | `{success, path}` |
| `edit_file` | 编辑现有文件 | `{path, operations[]}` | `{success, path}` |
| `read_file` | 读取文件内容 | `{path}` | `{content}` |
| `delete_file` | 删除文件 | `{path}` | `{success}` |
| `list_files` | 列出文件树 | `{directory?}` | `{files[]}` |
| `create_component` | 创建 React 组件 | `{name, code, propsSchema}` | `{success, id}` |
| `apply_theme` | 应用主题配置 | `{themeConfig}` | `{success}` |
| `install_dependency` | 安装 npm 依赖 | `{packages[]}` | `{success}` |
| `refresh_preview` | 刷新实时预览 | `{}` | `{status}` |
| `create_page` | 创建页面路由 | `{path, component}` | `{success, route}` |
| `create_api_route` | 创建 API 端点 | `{path, method, handler}` | `{success, endpoint}` |
| `run_linter` | 运行代码检查 | `{path?}` | `{errors[]}` |
| `deploy` | 触发部署 | `{environment}` | `{deploymentId}` |

## 2.3 Visual Editor（可视化编辑器）

| 子模块 | 功能 |
|---|---|
| 组件树面板 | 层级展示页面组件树、选中高亮、拖拽排序、显隐切换 |
| 画布区域 | 实时渲染、拖拽放置组件、框选、对齐辅助线 |
| 属性面板 | 编辑选中组件的 props、样式、事件绑定 |
| 响应式工具栏 | 桌面/平板/手机断点切换 |
| 代码同步 | Code ↔ Visual 双向绑定，编辑代码更新画布，编辑画布更新代码 |
| 撤销/重做 | 基于操作栈的 undo/redo |
| 右键菜单 | 复制/粘贴/删除/转为组件/包裹容器 |

### Code ↔ Visual 双向同步

```
Code → Visual:
  1. 解析 TSX/JSX 文件为 AST
  2. 提取组件树结构
  3. 渲染到画布

Visual → Code:
  1. 画布操作 → 更新组件树
  2. 组件树 → AST 修改
  3. AST → 代码生成（保持格式）

使用 magic-string 或 babel 进行精确代码修改（不重写整个文件）
```

## 2.6 AI Agent（智能体）

### 流程示例

```
Agent Flow 示例:

  [Trigger: Webhook] ──▶ [LLM: 分析需求] ──▶ [Tool: 创建文件] ──▶ [Tool: 部署]
                              │                                          │
                              ▼                                          ▼
                        [Condition: 是否需要数据库?]              [Notify: Slack]
```

### 节点类型

| 节点 | 说明 |
|---|---|
| Trigger | webhook / schedule / event / manual |
| LLM Node | 模型选择 + Prompt + 上下文 |
| Tool Node | 调用内置工具 |
| Condition | if/else 分支 |
| Loop | 循环 |
| HTTP Request | 外部 API 调用 |
| Transform | 数据转换 |
| Delay | 延迟 |
| Notify | 邮件/Slack/Webhook 通知 |

## 2.7 Multi Model（多模型）

| Provider | 模型 | 特点 |
|---|---|---|
| OpenAI | GPT-4o, GPT-4o-mini, o1 | 综合能力强、函数调用稳定 |
| Anthropic | Claude 3.5 Sonnet, Claude 3 Opus | 代码生成优秀、长上下文 |
| Google | Gemini 1.5 Pro, Gemini Flash | 多模态、速度快 |
| Groq | Llama 3.1 70B, Mixtral | 极速推理 |
| Ollama | Llama 3, Qwen, DeepSeek | 本地部署、隐私 |
| Mistral | Mistral Large, Codestral | 代码专精 |

### 特性

- 用户可自带 API Key（BYOK）
- 按任务类型智能路由（代码生成→Claude，快速对话→GPT-4o-mini）
- Token 用量与成本统计
- 模型能力探测（vision / function_calling / streaming）

## 2.8 Deploy（部署）

### 部署流程

```
用户点击部署
    │
    ▼
┌─ 构建项目 (sandbox service)
│   - npm install
│   - npm run build
│   - 生成静态文件 / SSR 配置
│
├─ 上传构建产物 (S3/MinIO)
│
├─ 调用部署平台 API
│   - Vercel: vercel deploy --prod
│   - Netlify: netlify deploy --prod
│   - Custom: Docker build & push
│
├─ 配置域名 & SSL
│   - 子域名: project.nova-studio.app
│   - 自定义域名: 用户绑定 CNAME
│
└─ 返回部署 URL + 日志
```
