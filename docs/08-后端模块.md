## 8.2 各模块详细设计

### AuthModule（认证模块）

```
职责: 用户认证、JWT 签发/验证、OAuth、权限校验

文件结构:
  modules/auth/
  ├── auth.module.ts
  ├── auth.controller.ts      # POST /register, /login, /logout, /refresh, /oauth/*
  ├── auth.service.ts         # 认证逻辑
  ├── strategies/
  │   ├── jwt.strategy.ts     # JWT 策略
  │   ├── jwt-refresh.strategy.ts
  │   └── github.strategy.ts  # GitHub OAuth
  ├── guards/
  │   ├── jwt-auth.guard.ts   # JWT 认证守卫
  │   └── roles.guard.ts      # 角色权限守卫
  └── decorators/
      ├── current-user.decorator.ts  # @CurrentUser()
      └── roles.decorator.ts         # @Roles('admin')

核心逻辑:
  - 注册: bcrypt 哈希密码 (cost=12)
  - 登录: 验证密码 → 签发 accessToken (15min) + refreshToken (7d)
  - 刷新: refreshToken → 新 accessToken
  - OAuth: code → provider token → 获取用户信息 → 创建/关联用户
  - 权限: JWT payload 包含 userId + orgId + role
  - API Key 加密: AES-256-GCM
```

```
  ├── agent.module.ts
  ├── agent.controller.ts    # Agent CRUD + 执行

  1. 解析 nodes + edges 为 DAG
  2. 从 trigger 节点开始

  4. 每个节点: 接收输入 → 执行 → 传递输出

  6. 记录每步执行结果到 agent_executions
```

### GitModule（Git 同步模块）

```
职责: GitHub OAuth、仓库关联、双向同步

  modules/git/
  ├── git.module.ts
  ├── git.controller.ts
  ├── git.service.ts         # 同步业务逻辑

  ├── pull.service.ts        # GitHub → VFS (拉取最新)
  └── branch.service.ts      # 分支管理

Push 流程:
  1. 读取 VFS 全量文件
  2. 与上次同步版本 diff
  3. 调用 GitHub API 逐文件创建/更新
  4. 记录 commit_hash
  5. 更新 github_syncs 记录

Pull 流程:
  1. 调用 GitHub API 获取仓库文件树
  2. 下载所有文件内容
  3. 更新 VFS (覆盖本地)
  4. 刷新预览
  5. 记录同步状态
```

## 8.3 服务间通信

| 通信方式 | 场景 | 技术 |
|---|---|---|
| HTTP REST | 客户端 ↔ BFF ↔ Core API | Axios / fetch |
| SSE (Server-Sent Events) | AI 流式响应 | EventSource |
| WebSocket | 实时协作、推送 | Socket.io |
| Redis Pub/Sub | 服务间异步事件 | Node Redis |
| gRPC (未来) | 服务间高性能调用 | @grpc/grpc-js |

```
部署流程:
  1. 接收部署请求 (projectId, platform, envVars)
  2. 从 VFS 读取项目文件
  3. 在隔离容器中 npm install + npm run build
  4. 上传构建产物到 S3
  5. 调用部署平台 API
  6. 配置域名 / SSL
  7. 返回部署 URL
  8. 发布 deploy:complete 事件
```
