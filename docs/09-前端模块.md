**双向同步:**

```
Code → Visual:
  1. 解析 TSX/JSX 文件为 AST
  2. 提取组件树结构
  3. 渲染到画布

Visual → Code:
  1. 画布操作 → 更新组件树
  2. 组件树 → AST 修改
  3. AST → 代码生成（保持格式）

使用 magic-string 或 babel 进行精确代码修改（不重写整个文件）
```

## 9.4 前端状态管理架构

```
┌──────────────────────────────────────────────────────────┐
│                   状态管理分层                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Server State (TanStack Query v5)                        │
│  ┌──────────────────────────────────────────────────┐    │
│  │ - 项目列表、文件、组件、主题、部署、GitHub       │    │
│  │ - 自动缓存、重新验证、乐观更新                    │    │
│  │ - Query Key 命名: ['org', orgId, 'projects']     │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│  Client State (Zustand)                                  │
│  ┌──────────────────────────────────────────────────┐    │
│  │ - workspace.store   (标签页/侧边栏/面板状态)      │    │
│  │ - editor.store      (文件标签/活动文件)            │    │
│  │ - chat.store        (对话/消息/流式缓冲)           │    │
│  │ - visual.store      (组件树/选中/画布)             │    │
│  │ - preview.store     (预览URL/状态/设备)           │    │
│  │ - theme.store       (主题配置/暗色模式)            │    │
│  │ - agent.store       (流程节点/边/执行)             │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│  Realtime State (WebSocket)                              │
│  ┌──────────────────────────────────────────────────┐    │
│  │ - 流式 AI 响应                                    │    │
│  │ - 文件变更推送                                    │    │
│  │ - 部署进度                                        │    │
│  │ - 协作光标/在线状态                               │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│  URL State (Next.js searchParams)                        │
│  ┌──────────────────────────────────────────────────┐    │
│  │ - 当前标签页 ?tab=chat                           │    │
│  │ - 活动文件 ?file=src/app/page.tsx                │    │
│  │ - 活动对话 ?conv=abc123                           │    │
│  └──────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────┘
```

### Query Key 命名规范

```typescript
// 组织
['organizations']                                    // 组织列表
['organizations', orgId]                             // 组织详情
['organizations', orgId, 'members']                  // 成员列表
['organizations', orgId, 'projects']                 // 项目列表

// 项目
['projects', projectId]                              // 项目详情
['projects', projectId, 'files']                     // 文件树
['projects', projectId, 'files', path]               // 单文件
['projects', projectId, 'conversations']             // 对话列表
['projects', projectId, 'components']                // 组件列表
['projects', projectId, 'themes']                    // 主题列表
['projects', projectId, 'deployments']               // 部署历史
['projects', projectId, 'agents']                     // Agent 列表

// 对话
['conversations', conversationId]                    // 对话详情
['conversations', conversationId, 'messages']        // 消息列表

// AI
['ai', 'models']                                     // 模型列表
['users', 'me', 'api-keys']                          // API Key 列表
```

### Zustand Store 设计原则

```
1. 单一职责: 每个 Store 管理一个功能领域
2. 不可变更新: 使用 immer 中间件
3. 持久化: workspace.store 的标签页状态持久化到 localStorage
4. 选择器: 使用 selector 避免不必要的重渲染
5. 与 TanStack Query 协同:
   - Store 管理 UI 状态（选中/打开/流式缓冲）
   - Query 管理数据（CRUD + 缓存）
   - Mutation 成功后 invalidate 相关 Query
```
